For Homebrew formula rendering, the release workflow should wait for npm metadata and use `npm view "@aictx/memory@${PACKAGE_VERSION}" dist.tarball` as the tarball URL, then download and hash that exact URL. Do not construct `https://registry.npmjs.org/@aictx/memory/-/memory-${PACKAGE_VERSION}.tgz` by hand; resolving `dist.tarball` avoids registry path assumptions and publish propagation races. After repairing a failed release, verify with `npm view @aictx/memory version`, `brew info aictx/tap/memory`, and `brew fetch aictx/tap/memory`.
