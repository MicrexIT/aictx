The tag-triggered release workflow checks whether @aictx/memory@PACKAGE_VERSION already exists on npm. If it does, the workflow skips the duplicate npm publish, then computes the SHA-256 from the existing npm tarball, renders Formula/memory.rb with scripts/render-homebrew-formula.mjs, and pushes the formula to aictx/homebrew-tap. If the package is not already on npm, the workflow publishes with provenance first. The workflow requires HOMEBREW_TAP_TOKEN in the npm GitHub environment with write access to the tap and checks out the tap before npm publish so missing access fails early.
