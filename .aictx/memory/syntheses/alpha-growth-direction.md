# Alpha growth direction

Working direction after alpha: position Aictx as the local-first memory layer for AI coding agents, not as another agent runtime. The near-term wedge should be the repeated pain of re-explaining project intent, conventions, workflows, traps, and decisions across Codex, Claude Code, Cline, OpenCode, and other agents.

Prioritize a tight adoption loop: one-command setup, agent-specific copy/paste integration recipes, a tiny demo repo or before/after transcript showing an agent loading memory and avoiding a real mistake, and docs that answer 'why not AGENTS.md only?', 'why not vector DB/RAG?', and 'why not just long context?'.

For community growth, create GitHub-facing proof and contribution paths: clear repo description/topics, screenshots/GIF of the viewer, CONTRIBUTING/CODE_OF_CONDUCT/SECURITY templates, good-first issues focused on docs/integrations/examples, and fast public issue response.

Product roadmap bias: improve onboarding and integrations before expanding storage complexity. Useful next surfaces include `aictx setup --apply` polish, published templates for AGENTS.md/CLAUDE.md/Codex, example `.aictx/` packs in real repos, migration/import from agent context files, and simple memory quality checks that make project memory feel governed and reviewable.

This is a working synthesis, not a finalized product commitment. Update it when product direction changes.
