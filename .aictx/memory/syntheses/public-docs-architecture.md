AICTX public documentation is implemented as an Astro Starlight site rooted at `docs/`, with public Markdown content under `docs/src/content/docs/`. Internal implementation/spec documents live under `specs/` and are not part of the public docs navigation or generated agent docs. The docs site is configured for `https://docs.aictx.dev`, generates `llms.txt`, `llms-full.txt`, and `llms-small.txt` through `starlight-llms-txt`, and deploys `docs/dist` to GitHub Pages via `.github/workflows/deploy-docs.yml`. The package ships public docs, and `aictx docs` reads the same bundled Markdown topics and can open hosted docs URLs. Public docs should use concise, neutral developer-tool documentation prose; generated/installable agent guidance remains separately directive for agent behavior.
