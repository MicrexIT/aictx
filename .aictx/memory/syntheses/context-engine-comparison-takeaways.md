# Context engine comparison takeaways

Aictx now incorporates the useful Unblocked/context-engine lesson as a lean, local-first memory-quality discipline: load context, work or encounter failure/correction, identify the memory gap, then repair durable project memory.

The implemented v1 shape stays inside existing object types. It adds optional organization facets (`domain`, `bounded-context`, `capability`, `business-rule`, and `unresolved-conflict`), renders active selected `conflicts_with` relations as `Memory conflicts to resolve`, keeps conflicted claims out of `Must know`, audits active syntheses without source provenance, and teaches suggestions/guidance to prefer source-backed syntheses and unresolved-conflict questions when relevant.

External connectors, hosted sync, embeddings, scanners, expert/social graphs, and new gap/probe/work item object types remain deferred until local durable memory discipline is strong. Domain-driven design can inspire organization, but Aictx should keep the language optional and plain.
