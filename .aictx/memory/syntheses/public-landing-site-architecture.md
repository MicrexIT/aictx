The public landing page site is implemented as a lean Astro app rooted at `site/`, separate from the Starlight docs app rooted at `docs/`. The site uses `site/src/pages/index.astro` for the landing page, `site/src/pages/blog/index.astro` for the blog index, `site/src/pages/blog/[...slug].astro` for generated post pages, and `site/src/content.config.ts` for the `blog` Markdown content collection under `site/src/content/blog/*.md`. Blog posts use frontmatter fields `title`, `description`, `publishedAt`, optional `updatedAt`, and optional `tags`; the blog index sorts posts newest first by `publishedAt`. Rendered Markdown post bodies are wrapped by `site/src/components/Prose.astro`, which uses Tailwind Typography classes. Public assets and `stars.js` live under `site/public/` so they are served from root-relative URLs on landing and blog routes. Root package scripts include `dev:site` and `build:site` for this Astro site.
