AICTX now implements the soft-role/lens/branch-handoff direction. The product model is: setup creates or repairs memory roles, load gives task-focused context, lens shows readable project views, remember saves durable discoveries, and handoff preserves unfinished branch state. Storage remains dynamic, with no mandatory placeholder files. The built-in role catalog powers setup coverage, lenses, viewer tabs, audit-oriented gaps, and docs; `synthesis.feature-map` remains the compatibility anchor for the Capability Map role. `aictx setup` is the normal onboarding command and applies conservative source-backed bootstrap memory by default, while `aictx init` remains the lower-level empty-storage initializer. `aictx lens` is CLI-only with project-map/current-work/review-risk/provenance/maintenance views. `aictx handoff` is CLI-only, stores one active `synthesis.branch-handoff-<branch>` object scoped to the current Git branch, and closes by marking it stale while optionally promoting durable project memory. Local MCP remains unchanged.
