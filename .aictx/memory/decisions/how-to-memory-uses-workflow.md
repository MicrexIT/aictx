Aictx represents durable project-specific how-tos with the existing `workflow` object type and `workflow` facet. Do not add a separate `how-to` object type or facet; use `workflow` for reusable procedures, runbooks, command sequences, setup, release, migration, debugging, recovery, verification, and maintenance routines. `aictx suggest --after-task` should prefer workflow memory and the workflow facet when task wording signals a reusable how-to.
