The local viewer should put canonical Aictx storage first. The main memory screen shows canonical object sections directly and no longer renders guided-view or load-preview/context-preview panels above the data. Lens/role coverage and `/load-preview` APIs remain available for CLI/API use. A dedicated Graph screen beside Schema Browser uses Cytoscape.js to visualize current active objects and active relations by default, with pan/zoom/fit/layout controls and an inspector for selected nodes and edges. The Svelte viewer imports `OBJECT_TYPES` and `FACET_CATEGORIES` from `src/core/types.ts` so viewer filters do not drift from the storage schema.
