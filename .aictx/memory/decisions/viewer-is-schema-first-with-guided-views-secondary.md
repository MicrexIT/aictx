The viewer should represent canonical Aictx storage first: object `type`, `status`, `facets.category`, scope, evidence, and relations. Guided context from `bootstrap.lenses` and `role_coverage` is rendered as a separate secondary layer rather than replacing canonical type/facet grouping. The Svelte viewer imports `OBJECT_TYPES` and `FACET_CATEGORIES` from `src/core/types.ts` so viewer filters do not drift from the storage schema.
