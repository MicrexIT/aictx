The local memory viewer keeps the existing token-protected viewer API contract, opens the current project context first, and presents memory as an agent-first context contract with humans as secondary maintainers. The page hierarchy is a clean document flow rather than a dense dashboard: hero, Agent Brief, Context Quality, Diagnostic Map, optional compact Standalone Review, Selected Memory under the map, then Memory Library. Avoid repeated decorative icons in the mobile header and hero; on mobile the header icon is enough and the hero icon should be hidden. The viewer uses human-readable Git status labels such as `Uncommitted memory` instead of internal labels like `Memory dirty`, and the badge title explains that uncommitted memory differs from committed team state. The core map renders active relation endpoints in Relations mode with deterministic preset lane positions instead of a force layout; Relations exposes the count of active relation edges. Standalone mode exposes the count of current unlinked active memories, keeps unlinked memories separate from the relation layout, and explains that unlinked memory is not automatically bad. Keep standalone review lightweight on the main page: agents handle obvious links during normal work, humans review ambiguous/high-impact changes, and suggested `aictx remember --stdin` patches should be hidden behind disclosure rather than shown as a dominant command block. Memory list rows should use an `Inspect` affordance, not `Open`; selecting a row updates the single Selected Memory panel and scrolls it into view so the action has visible feedback. Memory list filters are scoped to the memory library below the map rather than appearing to control the map. Selecting a graph node changes only the selected styling and detail panel; it must not invert or recompute the map layout. Long graph labels are compacted into short multiline labels, the visible chip/tag dump is hidden as an accessible summary, and the graph remains backed by accessible DOM summaries for tests and screen readers. On mobile widths, the left rail becomes a sticky compact top header with a `Menu` button; the menu must start closed and open as a polished dropdown sheet below the header, with search, vertical page rows, and grouped export controls inside the sheet. Mobile sections stack into one column so agents and humans can inspect the brief, selected memory, quality cards, map, standalone guidance, and memory library without horizontal page overflow.
