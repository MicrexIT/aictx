Cline Memory Bank documentation describes a structured markdown methodology centered on a memory-bank/ directory with six core files: projectbrief.md, productContext.md, activeContext.md, systemPatterns.md, techContext.md, and progress.md. Its instructions tell Cline to read all memory bank files at the start of every task, initialize missing files, update them after significant milestones or context pressure, and keep files concise. Cline treats Memory Bank as project knowledge while checkpoints protect code state. The public docs also allow additional context files for complex features, integrations, APIs, testing strategies, and deployment procedures.
