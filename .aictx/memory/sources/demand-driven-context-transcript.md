The talk argues that enterprise agent failures often come from poor institutional knowledge, not model quality or retrieval alone. Its proposed loop uses real work items such as incidents or Jira tickets to make agents fail, extract the missing knowledge checklist, ask domain experts for the missing facts, and curate reusable context blocks. At scale it suggests a context gap scanner that generates probes from archived work items, scores clean/stale/incomplete/missing/tribal knowledge, deduplicates findings, prioritizes critical gaps into a Kanban-style board, and stores curated context in a Git-reviewed repository. The talk also recommends an optional meta model for systems, APIs, business processes, terms, and their relationships so agents have a map of the knowledge base.
