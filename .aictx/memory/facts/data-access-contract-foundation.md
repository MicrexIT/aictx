# Data-access contract, adapter wiring, and profiles exist

T007 added the host-neutral data-access service foundation under `src/data-access/`. The default service exposes `load`, `search`, `inspect`, `diff`, and `applyPatch`, accepts `DataAccessProjectTarget` as either `cwd` or `project-root`, resolves that target to one project root and `.aictx` boundary, and returns current `AppResult` envelopes.

T008 routed the CLI and MCP read adapters for load, search, inspect, and diff through `dataAccessService`. CLI read commands pass `{ target: { kind: "cwd", cwd } }` into the shared service before existing rendering, and MCP read tools do the same after resolving optional `project_root`. Existing CLI JSON output, MCP structured content, search ranking, and context compilation remain shared through the app/data-access layers.

T009 routes structured patch saves from CLI `aictx save --stdin`, CLI `aictx save --file`, and MCP `save_memory_patch` through `dataAccessService.applyPatch`. The save adapters pass `{ target: { kind: "cwd", cwd }, patch }` into the data-access service; existing app/storage save orchestration still handles patch validation, secret checks, project locking, dirty touched-file backup, repair/quarantine, event append, and index update. MCP still exposes only `save_memory_patch` as the write primitive.

T010 added `src/data-access/adapter-profiles.ts` as internal host adapter profile metadata exported from `src/data-access/index.ts`. The default active `local-mcp` profile maps `load_memory`, `search_memory`, `inspect_memory`, `save_memory_patch`, and `diff_memory` to `load`, `search`, `inspect`, `applyPatch`, and `diff`. The `future-generic` profile is inactive and maps generic `search` to data-access `search` and generic `fetch` to data-access `inspect`; it is returned only by explicit selection, and active local MCP registration in `src/mcp/server.ts` remains unchanged.

Focused unit coverage lives in `test/unit/data-access/service.test.ts` for project targeting, nested cwd boundary resolution, and envelope preservation. Adapter profile coverage lives in `test/unit/data-access/adapter-profiles.test.ts` for local MCP mappings, future generic mappings, default inactivity, and unsupported profile IDs. Read parity coverage lives in `test/integration/mcp/read-tools.test.ts` and `test/integration/cli/read-only.test.ts`. Write parity coverage lives in `test/integration/cli/save.test.ts` and `test/integration/mcp/save-tool.test.ts`; the CLI save test spies on `dataAccessService.applyPatch` for stdin/file input, and the MCP save test compares canonical output across CLI stdin, CLI file, and MCP save.
