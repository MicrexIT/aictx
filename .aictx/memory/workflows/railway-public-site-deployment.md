Deploy the public landing/blog site from the repository root because `package.json` and `pnpm-lock.yaml` live at `/`. Railway config-as-code is now defined in `railway.toml`: Railpack builds with `pnpm build:site` and starts with `pnpm serve:site`. `pnpm build:site` runs `astro build --root site` and emits static assets to `site/dist`; `pnpm serve:site` runs `scripts/serve-site.mjs`, a Node static server that binds to `process.env.PORT` and serves `site/dist`. Keep Railway Root Directory as `/`; do not set it to `site` unless a site-local package manifest and lockfile are added. The root `engines.node` requires Node >=22, which Railpack reads. The blog uses Astro content collections and static paths; an empty `site/src/content/blog` builds with a warning but is not a deployment failure.
