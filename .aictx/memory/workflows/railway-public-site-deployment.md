Deploy the public landing/blog site from the repository root because `package.json` and `pnpm-lock.yaml` live at `/`. Railway config-as-code is defined in `railway.toml`: Railpack builds with `npm install -g pnpm@10.11.0 && pnpm build:site` so the build does not assume pnpm is already available, and deploy starts with `node scripts/serve-site.mjs` so runtime does not depend on pnpm. `pnpm build:site` runs `astro build --root site` and emits static assets to `site/dist`; `scripts/serve-site.mjs` binds to `process.env.PORT` and serves `site/dist`. Keep Railway Root Directory as `/`; do not set it to `site` unless a site-local package manifest and lockfile are added. The root `package.json` declares `packageManager: pnpm@10.11.0` and `engines.node >=22`. The blog uses Astro content collections and static paths; an empty `site/src/content/blog` builds with a warning but is not a deployment failure.
