Aictx contributors should work from a feature branch, not push directly to `main`. Start from latest `main`, create a short topic branch, run relevant verification such as `pnpm typecheck` and `pnpm test`, commit only intended files, push the branch, and open a pull request targeting `main`. Invited write collaborators may self-merge after the required GitHub checks pass: `Node 22`, `Node 24`, and `Analyze JavaScript and TypeScript`. If GitHub says the PR branch is behind `main`, update the branch and wait for checks again. CONTRIBUTING.md includes a copyable prompt for AI agents that manage git, including the Aictx load/save memory discipline.
