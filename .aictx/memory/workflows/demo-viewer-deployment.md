The public viewer demo is available at `https://demo.aictx.dev/`; the viewer automatically uses the public `demo` token on that host. The compatibility URL `https://demo.aictx.dev/?token=demo` also works. The demo is a read-only Cloudflare Worker deployment. The Worker in `src/viewer/demo-worker.ts` serves the built Vite viewer assets through Cloudflare Workers Static Assets and handles seeded demo API routes.

Seed data is generated by `scripts/build-viewer-demo-data.mjs` as deterministic synthetic memory for a generic Todo App project with id `project.todo-app`; it is not derived from this Aictx repository's own `project.aictx` memory. The generated `src/viewer/demo-data.generated.json` should include canonical object types, facet categories, provenance relations, non-empty `role_coverage`, and non-empty `lenses`. Unit tests should reject the old `project.aictx` seed.

Cloudflare Git integration owns the public demo deployment. Do not add a GitHub Actions Wrangler deploy for `demo.aictx.dev`; that duplicates Cloudflare's deployment owner and requires GitHub `CLOUDFLARE_API_TOKEN`/`CLOUDFLARE_ACCOUNT_ID` secrets. Use `pnpm build:demo-viewer` locally or in validation jobs to build viewer assets, regenerate the seed, and typecheck the Worker. Keep `pnpm deploy:demo:cloudflare` only as a manual Cloudflare/Wrangler escape hatch when an authenticated local environment is available.

The public `demo` token is compatibility routing, not real authentication. Demo mode hides Obsidian export in the Svelte viewer and the Worker returns non-success JSON for write/export routes.
