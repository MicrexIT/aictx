Deploy the public landing/blog site to Vercel from the repository root. The root `vercel.json` sets `installCommand` to `npm install -g pnpm@10.11.0 && pnpm install --frozen-lockfile`, `buildCommand` to `pnpm build:site`, and `outputDirectory` to `site/dist`. `pnpm build:site` runs `astro build --root site`, producing the static landing page and blog output. The Vercel project is linked locally as `rivista-rinascita/aictx` but `.vercel/` is ignored. Production deployments are available through `https://aictx-five.vercel.app/`; `/` and `/blog/` were verified on 2026-05-05. Vercel custom domains `aictx.dev` and `www.aictx.dev` were added to the project on 2026-05-05. With the current GoDaddy nameservers (`ns25.domaincontrol.com`, `ns26.domaincontrol.com`), Vercel CLI reports these required DNS records: `A aictx.dev 76.76.21.21` and `A www.aictx.dev 76.76.21.21`. Alternatively, change nameservers to Vercel (`ns1.vercel-dns.com`, `ns2.vercel-dns.com`).
