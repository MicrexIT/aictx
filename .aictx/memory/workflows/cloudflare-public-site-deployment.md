Deploy the public landing/blog site to Cloudflare Workers from the repository root using Workers static assets. The root `wrangler.jsonc` names the Worker `aictx`, sets `compatibility_date` to `2026-05-05`, and serves static assets from `./site/dist` with `html_handling: auto-trailing-slash` and `not_found_handling: 404-page`. Cloudflare Workers Builds should use build command `pnpm build:site` and deploy command `npx wrangler deploy` (or the package script `pnpm deploy:site:cloudflare`). The repo pins `wrangler` as a dev dependency and has `.node-version` set to Node 22. `pnpm build:site` and `pnpm exec wrangler deploy --dry-run` were verified on 2026-05-05. For `aictx.dev`, prefer Worker Custom Domains over plain Routes: a dashboard Route value of `aictx.dev` did not create an apex A/AAAA record and only matched the root path during forced-edge testing; `/blog/` returned Cloudflare 530. Use Settings > Domains & Routes > Add > Custom Domain for `aictx.dev` (and separately `www.aictx.dev` or a redirect). If using Routes instead, the hostname must have a proxied DNS record and the route pattern must include paths such as `aictx.dev/*`. Do not deploy this project under the unrelated Vercel `rivista-rinascita` scope; the Vercel project was deleted and is superseded by Cloudflare.
